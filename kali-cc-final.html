05 — DANGER ZONE
Where Claude Code draws the line
Even with --dangerously-skip-permissions and custom CLAUDE.md, there are hard limits Claude will not cross. Knowing these helps you plan your workflow.
🚫
Writing malware or weaponized exploits
Claude will analyze malware and explain how it works — but won't write functional ransomware, rootkits, RATs, or weaponized zero-day exploits from scratch. It can write PoC code for known vulnerabilities in authorized testing contexts.
🚫
Social engineering / phishing campaigns
Won't craft phishing emails, pretexting scripts, or deepfake content designed to trick real people. It will explain social engineering techniques educationally but won't create ready-to-deploy attack materials.
🚫
Attacking unauthorized targets
If Claude detects you're targeting real infrastructure without clear authorization signals (HTB VPN, bug bounty scope docs, localhost), it will push back. CLAUDE.md persona helps here — explicitly state targets are authorized.
🚫
Bypassing authentication on live systems
Claude will help you understand auth bypass techniques and test them on your own apps, but resists helping break into accounts or systems you don't own — even with skip-permissions enabled.
⚠️
Gray zone — depends on context
Exploit chains for known CVEs, password cracking, privilege escalation, network pivoting — Claude will do these if the context is clearly authorized (HTB, CTF, bug bounty, your own infra). The CLAUDE.md security persona is what makes the difference.
✅
Always allowed
Scanning your own systems, CTF/HTB challenges, forensic disk analysis, OSINT from public sources, vulnerability research, code auditing, security report writing, explaining attack techniques educationally.
💡
The workaround: Claude's limits are about intent, not capability. If your CLAUDE.md clearly establishes authorized testing context, and you're using --dangerously-skip-permissions in an isolated lab, Claude will go deep on exploitation, privilege escalation, and post-exploitation. The skills packs from Transilience push this even further with 46 attack methodologies baked in.
